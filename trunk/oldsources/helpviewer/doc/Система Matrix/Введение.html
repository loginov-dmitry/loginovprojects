Введение

Система легко интегрируется с любым проектом, написанным на Delphi, и имеет огромный диапазон применений, начиная с простейшего хранения элементов массива и заканчивая такими сложными вычислениями, как вейвлет-преобразования и нейронные сети.

Бесконечная вложенность вычислений дает возможность многократного вызова подпрограмм из любого места программы, что достигается благодаря технологии изолирования рабочих областей.

Система имеет ядро, находящееся в файле Matrix.pas и набор прикладных подпрограмм, расположенных в других pas-файлах. Ядро системы содержит набор базовых подпрограмм, достаточных для реализации любого вычислительного алгоритма, в том числе имеются функции для хранения массивов в файлах и чтения из файлов, а также набор функций, обеспечивающих взаимодействие с системой Matlab.

При создании системы Matrix ставилось целью реализовать базовые операции и функции системы Matlab и достигнуть аналогичной скорости их выполнения. В результате получился функционально законченный продукт, сходный с системой Matlab по своим возможностям и ничем незаменимый для программиста, работающего в среде Delphi.

Система Matrix становится уникальной по своим возможностям при конвертировании алгоритмов, написанных на языке Matlab в код среды Delphi. Данная задача действительно упрощается во много раз, так как с помощью системы Matrix программист работает с терминами, очень похожими на термины системы Matlab.

Matrix является свободным продуктом, что обеспечивает экономическую выгоду при его использовании по сравнению с дорогостоящей системой Matlab.

Одно из главных преимуществ системы Matlab в том, что для нее написано необозримое количество m-функций, используемых во всех отраслях науки. То же самое можно делать с помощью системы Matrix. Вместе с ядром системы распространяются несколько pas-файлов, которые являются сборниками подпрограмм (далее по тексту вместо этого термина вы можете встретить словосочетание "внешние функции"). Среди них находится модуль Signals.pas, содержащий работоспособные подпрограммы по обработке сигналов (эти подпрограммы применяются в области кардиологии для обработки сигналов ЭКГ). Вы можете писать аналогичные сборники подпрограмм. Посылайте свои разработки на электронный ящик matrix@kladovka.net.ru и они будут включены в состав Matrix.

В ходе программирования ядра Matrix основной упор делался на ускорение работы функций. В результате скорость обработки матриц в Matrix стала практически такой же, как в Matlab.

Matrix содержит мощные средства для работы с файлами. При необходимости вы можете сохранить массив (или указанную рабочую область) в файлы следующих форматов:
- ascii-формат - хранит элементы массива в текстовом виде. Название файла соответствует имени массива;
- текстовый формат - может хранить неограниченное число массивов в текстовом виде, причем запись одного массива состоит из имени массива, символа "=" и списка элементов.
- двоичный формат - позволяет хранить неограниченное число массивов в двоичном виде. При использовании данного формата скорость работы с файлами самая высокая, а размер файла получается минимальным.

Система Matrix включает в себя средства для взаимодействия с системой Matlab. Существуют задачи, которые невозможно решить с помощью Matrix, однако их можно выполнить средствами Matlab. В этом случае вы просто передаете в Matlab необходимые массивы, выполняете необходимые операции и считываете из среды Matlab выходные данные.

Кроме того, в систему Matrix включены средства для взаимодействия с 2-мерными массивами среды Delphi. Вы можете скопировать элементы массива любого типа в массив формата системы Matrix или выполнить обратную операцию. Данное средство используется в основном при работе с базой данных и при редактировании старых программ. В последнем случае вам может потребоваться заменить устаревший или ненадежный алгоритм на код системы Matrix, но нельзя менять формат представления данных, т.е. входные и выходные данные - это обычные массивы среды Delphi. Вы просто копируете исходные массивы в массивы Matrix, программируете новый алгоритм и копируете полученный массив Matrix в массив среды Delphi.

В ядре системы находится большое количество вычислительных функций, в их число входят функции, предназначенные для:
- работы с многочленами;
- решения СЛАУ;
- матричного умножения и другие функции, о которых вы узнаете в следующих разделах.

Что касается совместимости с операционной системой, то отметим, что при написании системы Matrix не использовались нестандартные API-функции, поэтому применение Matrix не может вызвать проблем с совместимостью.

Каждый элемент массива в Matrix занимает 8 байт (тип Double), поэтому необходимо учитывать возможности компьютера, на котором будет работать приложение, использующее Matrix. Вы можете создавать массивы с любым числом элементов, однако при нехватке памяти появится сообщение об ошибке. Система проверяет объем свободной виртуальной и оперативной памяти и никогда не будет пытаться увеличить размер файла подкачки.

Система Matrix во время работы приложения не обращается к жесткому диску, поэтому приложение, запущенное с оптического диска, будет работать без ошибок. Единственное, что система может записать на диск - это массив или рабочую область. В двоичных файлах массивы занимают столько же места, сколько и в оперативной памяти (1 элемент - 8 байт).

Скорость вычислений в Matrix примерно та же, что и в Matlab. В ходе программирования ядра Matrix использовались самые быстрые алгоритмы, поэтому ни один программный продукт, сходный по своим функциям с Matrix принципиально не может работать заметно быстрее. Использование Matrix - это самый удобный и быстрый способ решения поставленной задачи, однако код будет работать быстрее, если вы сможете правильно (используя, где можно, самые быстрые функции) переписать его на чистый язык Delphi. Для этого вы должны быть профессиональным программистом, т.к. оптимизация - задача не из легких, а новичку такая задача не по силам.

Применяя Matrix, даже начинающий программист сможет решать сложнейшие задачи, которые раньше требовали большого опыта и профессионализма. С появлением Matrix время разработки программного продукта уменьшается в десятки раз, значительно улучшается качество кода и структура программы, увеличивается надежность приложения. В дальнейшем такой код становится гораздо проще редактировать.
